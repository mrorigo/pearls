Issue Tracking for the Next Generation of Software Delivery

Pearls is a Git-native issue tracker that keeps planning close to code. Structured, dependency-aware, and local-first, it gives humans and agents one reliable source of truth.

Local-First by Design

Work offline, commit issue state with code, and keep full history in your repository.

Graph-Aware Execution

Dependencies and FSM transitions keep execution order clean for people and agents.

Trustworthy Operations

Atomic writes, lock coordination, merge conflict handling, and integrity checks.

Why Teams Adopt Pearls

Cloud trackers add context switching and API overhead to every automation loop. Pearls stays in your repo and makes issue state a first-class development artifact.

Agent Friendly Safe Merges Flexible Metadata Standalone

Production-Ready Workflow

`prl init` sets up `.pearls`, Git hooks, and merge integration. `prl sync`, `prl doctor`, and `prl compact` complete a practical end-to-end operating model.

prl hooks prl merge prl doctor prl sync

Install Pearls

Get the `prl` binary in your PATH with a single command.

cargo install --git https://github.com/mrorigo/pearls pearls-cli
